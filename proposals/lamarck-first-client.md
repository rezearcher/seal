# Proposal — Lamarck as Seal's first client (and a [PROVEN] route for memory-poison defense)

**Status:** direction note (Rez, 2026-06-11). Forward-looking; no Seal code change required yet.
Companion note in `~/projects/lamarck/docs/seal-integration.md`.

## The opportunity
**Lamarck** (`~/projects/lamarck`) is an always-on population of tiny neural "cells" that learn from a
data stream and pass acquired skill to offspring — a continuously self-training system. Its **#1
documented existential risk is model collapse via recursive self-training**: if a cell's fabricated
output becomes another cell's training/grading signal, the population rots. Its safety rule is
"store/replay only **authentic exogenous** observations; the grader stays exogenous."

**That rule is a memory-poisoning defense — which is exactly Seal's `sign_memory` /
`verify_on_recall` / VPE provenance story.** Lamarck wants to adopt Seal as an *early primitive* to
make that invariant a cryptographic guarantee rather than a hope. Lamarck becomes Seal's **first
client**, and a real dogfooding loop.

## Why this matters for Seal specifically
Seal's `THESIS_AND_VALIDATION.md` tags the memory-poison / sign-and-verify story largely **[THEORY]**.
Lamarck offers a **falsifiable, automated test of effectiveness**:

> Run Lamarck two ways — (control) episodic memory unsigned, vs (treatment) every entry `sign_memory`'d
> with exogenous provenance and `verify_on_recall` enforced before any cell can retrieve/replay it.
> If the control collapses (diversity/novelty rot, the known failure) and the treatment does not, that
> is a reproducible **[PROVEN]** result for Seal's memory-poison defense — the kind of concrete
> evidence that sells to clients, generated by a real system rather than a synthetic red-team.

This is "prove its effectiveness and make it more effective for clients," with a live workload.

## Feature requirements this client surfaces
1. **Sign arbitrary payloads, not just text.** `sign_memory(content: str)` and `vpe_sign(prompt: str)`
   are string-typed. Lamarck's "memory" is a **byte-window / structured event**, not a prompt.
   - Near-term: clients base64/serialize to a string (works, but lossy/awkward).
   - Proposed: a first-class **bytes / structured-record envelope** API (`sign_bytes` / `sign_record`)
     so non-prompt clients are first-class. *(Lamarck is the forcing function.)*
2. **Verify-at-scale throughput.** Lamarck verifies on **recall/replay**, potentially many ops/sec
   across a population. Publish a clear Ed25519 verify throughput number (extend
   `benchmark_vpe_verify.py`) and a guidance note on "sign-on-write, verify-on-read at N ops/sec."
3. **Provenance for a non-human issuer.** The "issuer" of a Lamarck memory is the **stream ingestion
   source / the world**, not `user:rez`. Confirm/representation for a *machine/world* issuer identity
   and scope (e.g. `issuer="world:byte-stream"`, scope = which exogenous source).
4. **Library packaging for an in-process dependency** — Lamarck would `pip install -e` Seal (or pin a
   git version). Confirm the import surface (`from seal.memory import sign_memory, verify_on_recall`)
   is stable API.

## RESULT — memory-poison defense [PROVEN] in a live system (2026-06-11)
First-client validation executed. In a 1000-tick Lamarck run on a real text corpus, 50% of episodic
writes were fabricated (constant garbage target = self-training poison):
- **Unsigned store (no Seal):** all 16087 poison events admitted; replay corrupted learning; the
  population's prediction loss on *real held-out data* rose to **5.84 nats — past the 5.55-nat uniform
  prior** (worse than random guessing). **Collapse.**
- **Seal-gated store** (`sign_memory(writer="world:stream")` on write, `verify_memory(trusted_writers=
  {"world:stream"})` on admission; poison forged with a non-world key → invalid signature):
  **16087 / 16087 poison events rejected**, fresh-loss 5.48 nats ≈ the clean baseline (5.29).
- Collapse magnitude +0.55 nats; Seal-protected +0.19 nats.

**This moves Seal's memory-poison claim from [THEORY] to [PROVEN]** — evidence from a real always-on
self-training workload, not a synthetic red-team. Notable: the collapse showed ONLY in the exogenous
grader; the unsigned population's *diversity even rose* (cells diverging on garbage) — so a diversity
check would miss it and provenance verification is the correct defense. Repro:
`~/projects/lamarck/experiments/m1_seal_collapse/` (`collapse_test.py`, `RESULTS.md`). Suggest adding a
row to `THESIS_AND_VALIDATION.md`'s ledger citing this.
